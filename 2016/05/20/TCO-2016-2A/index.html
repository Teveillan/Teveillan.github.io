Topcoder Open 2016 Round 2A

發表於 5月 20 2016   |   分類於 红名之路   |  

学算法学到后期简直就是在学玄学。

叉姐有一份题解：http://async.icpc-camp.org/d/420-topcoder-open-2016-round-2a

Easy: LCMGCD

最初有1~100一百个数字。 选定一个数N(<=100)。
A可以从其中取出K(<= 15)个数字，把其他数字扔掉。
B可以从剩余的数字中再扔掉一些数，然后再给剩余的一些数中取负号，使得最后的和为N。
问A该怎么取才能使得B一定得不到和为N的结果。

推论：当N不能被d整除时，如果我们取的数都为d的倍数，则和一定不为N。

当[2,6]中有一个数d不为N的因子时, 从小到大选取K个d的倍数。
不然此时N必为60的倍数，因为N<=100，所以N＝60。如果选15个7的倍数，则最大值105>100，超出范围。此时选14个7的倍数，再选一个1就好。

Medium: CliqueCuts

这个解法应该算peter的改进版。简直就是玄学。

题意：
有n(<=45)个点，无向图。
每次选一个点的集合A，保证集合中的点为团（任意两点之间都有边）。
则B ＝ V - A。
对于任意一条边， 权重为c：

  • 如果两个端点都在A中，则res += c
  • 如果两个端点都在B中，则res -= c

问对于所有的{A, B}，res之和为多少。

记忆话搜索。
对于每个点的邻边我们用状态压缩存起来。
V<=23的部分用记忆化，这部分的复杂度$O(2^{23})$
V >23的部分用dfs，这部分复杂度$O(2^{22})$
再枚举两个端点，此时已经能求解了。

玄学部分
对于一个集合{A, B}:

  • 对于两个端点都在A中的边, res += c * 1
  • 对于一个端点在A中，一个端点在B中的边, res += c * 0
  • 对于两个端点都在B中的边, res += c * -1

这三个系数分别为{1, 0, -1}, 如果此时我们给每个值+1，变为{2, 1, 0}，我们发现了什么？
此时的系数与端点在A中的个数一样！！
也就是说，假如一条边的两个端点都在A中，则这条边权加两次。
如果只有一个端点在A中，只加一次。
否则一次都不加。

这样的话便不用一条一条边的求解。
我们记录每个点在团中出现的次数$num_i$, 邻边的权重之和$cost_i$，则$res += num_i * cost_i$

最后，令A的总数为$num$, 总边权和为$cost$，则$res -= num * cost$。这就是最终解。
